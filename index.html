01 / featured

Flatio — Home Rental Booking Web Application

The problem: Finding and booking rental homes online in most markets means juggling WhatsApp messages, PDF listings, and manual payment confirmations. I wanted to build the end-to-end experience properly.

What I built: Flatio is a full-stack home rental platform where landlords can list properties and tenants can browse, view details, and book — all in one place.

Key technical decisions: I chose Node.js + Express for the API layer for its speed and flexibility, and structured the booking logic to handle date conflict detection server-side to prevent double bookings.

What I learned: Designing a booking system taught me to think carefully about race conditions and data integrity — two reservations hitting the same dates at the same time is a real problem that needs to be solved at the database level, not just the UI.

02 / featured

MPESA B2C Bulk Payments System

The problem: Businesses running bulk M-Pesa payments — salaries, supplier payouts, agent disbursements — often rely on manual processes or expensive ERP integrations. There was no simple, standalone tool for finance teams to upload a spreadsheet and push payments directly.

What I built: A secure web portal where authorized users upload an Excel file of payment recipients, the system validates the data, and triggers real M-Pesa B2C transactions via the Safaricom Daraja API — with real-time status tracking and a reconciliation report at the end.

Key technical decisions: Sensitive data (API credentials, phone numbers) is encrypted at the database level, not just in transit. Role-based access ensures only authorized users can initiate payments. The Excel validation layer catches formatting errors and duplicate entries before any transaction is fired.

What I learned: Working directly with a financial API like Daraja taught me that error handling and reconciliation matter more than the happy path — a failed transaction needs a clear paper trail, not just a generic error message.
